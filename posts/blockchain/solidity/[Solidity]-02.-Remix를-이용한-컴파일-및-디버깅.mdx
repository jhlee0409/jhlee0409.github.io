# 서론

- `Solidity`는 구문이 끝나면 `;` 을 붙여주어야한다.
- 붙이지 않고 컴파일 할 시 어떤 에러를 띄워주는지 알아보자.

# 컴파일 하기

- `Remix` 좌측 메뉴에서 컴파일 버튼을 눌러준다.

![Remix 좌측 메뉴](/assets/posts/blockchain/solidity/Solidity-02/01.png)

- 그럼 아래와 같은 화면이 나온다.
- 기본적인 설정은 둔 채 `CONTANCT`에서 컴파일할 계약을 선택한 후  컴파일 한다.

![Remix 컴파일 페이지](/assets/posts/blockchain/solidity/Solidity-02/02.png)

# 에러 테스트

> e.g. `storeData`에 `;` 을 제거한 후 컴파일 해보자

![;을 제거해보자](/assets/posts/blockchain/solidity/Solidity-02/03.png)


- `Remix`에서 컴파일 했을 시에 결과 화면이다.
![Remix 컴파일 이후](/assets/posts/blockchain/solidity/Solidity-02/04.png)


- `line:21`에 에러 표시가 뜨지만 해당 `line`의 코드는 문제 없다.
- 에러 메세지를 확인해보자
- `;` 이 올걸 예상했지만 `function` 이 있다는 얘기다.
- 이는 `line:18`에서 `;` 을 붙여야만 해당 구문이 끝났다고 이해를 하는데 예상치도 못한 코드가 이어졌기 때문이다.

![파싱 에러](/assets/posts/blockchain/solidity/Solidity-02/05.png)


# 해결

- `line 21`에서 `;`을 추가 해주면 해결!
![; 추가 이후 컴파일](/assets/posts/blockchain/solidity/Solidity-02/06.png)


# 결론

- 에러를 발생시키는 `line`이 아닌 그 다음 구문에서 에러를 띄워주기 때문에 혼란스러울 수 있다.
- 에러를 띄워주는 `line` 이전 코드들을 확인해보는 습관을 들이자.
